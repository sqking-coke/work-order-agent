# 智能工单处理Agent - 项目设计文档

## 1. 项目概述

### 1.1 项目背景

企业日常运营、客服运维、系统保障场景中，会产生大量用户咨询、故障反馈、业务申诉、运维报错等工单数据。传统工单处理完全依赖人工流转，存在诸多痛点：工单堆积严重、人工分类效率低、重复问题高频出现、流转分配不规范、问题响应滞后、历史工单经验无法复用。

传统工单系统仅实现工单录入、人工流转、状态修改等基础CRUD能力，**无智能识别、自动分派、智能答疑、根因分析、自动复盘能力**，大量简单重复性工单消耗人力成本，复杂工单无法快速定位问题。

基于以上痛点，本项目实现**Java原生智能工单处理AI Agent**，摒弃重型AI框架，基于后端工程化思想，完成「工单智能分类、实体信息抽取、自动责任分派、知识库智能答疑、复杂工单流转、异常根因分析、工单复盘总结」的全自动化闭环，实现工单处理降本增效、标准化、智能化。

### 1.2 项目定位

面向企业客服、运维、业务中台的**智能工单自动化AI Agent实战项目**，贴合企业OA、客服系统、运维工单系统真实业务场景。

区别于传统工单管理系统，本项目核心是**AI Agent智能决策+业务流程自动化**：

- 摆脱人工干预，AI自动完成工单意图识别、分类分级、信息提取
- 结合业务规则+大模型能力，实现简单工单自动闭环、复杂工单自动流转
- 纯Java工程化架构，流程可控、权限可控、数据安全可控，适配企业落地要求

### 1.3 核心目标

- 实现工单全流程智能化处理，替代人工分类、分派、答疑、复盘等重复工作
- AI自动完成工单意图识别、优先级分级、问题类型分类、关键信息抽取
- 结合轻量化RAG知识库，实现常见问题自动回复、简单工单自动办结
- 复杂工单自动匹配对应责任部门/责任人，触发标准化流转流程
- 支持工单超时预警、异常工单识别、周期性工单复盘、问题汇总分析
- 实现工单处理全流程可溯源、可统计、可复盘

## 2. 需求分析

### 2.1 功能需求

#### 2.1.1 工单基础管理能力

- 支持工单新增、编辑、状态修改、关闭、撤销等基础操作
- 支持工单多维度筛选：类型、优先级、处理状态、创建时间、责任人
- 支持工单标签管理、问题归类、业务场景划分

#### 2.1.2 AI工单智能解析能力

- 工单文本意图识别：咨询类、故障类、申诉类、建议类、运维报错类
- 智能优先级分级：紧急、高、中、低四档自动判定
- 关键实体抽取：用户信息、问题模块、报错关键词、业务场景、设备信息
- 自动识别重复工单、相似工单，避免重复处理

#### 2.1.3 智能分派与流程流转能力

- 基于工单类型、业务模块，自动匹配对应责任部门与处理人
- 简单咨询类工单AI自动回复、直接办结，无需人工介入
- 故障、运维类工单自动进入流转流程，等待人工处理
- 支持工单超时未处理预警、自动催办、流转节点记录

#### 2.1.4 知识库RAG智能答疑能力

- 支持上传业务文档、FAQ手册、运维手册、常见问题库
- 自动切片、语义检索，结合大模型生成精准答疑内容
- 知识库动态更新，持续优化答疑准确率

#### 2.1.5 工单统计与AI复盘能力

- 统计工单总量、办结量、待处理量、超时量、重复工单量
- AI自动汇总周期内高频问题、核心故障、薄弱业务模块
- 自动生成工单处理周报、月报、问题整改建议

#### 2.1.6 会话记忆与工单调优能力

- 支持用户多轮对话答疑，关联历史工单记录
- 支持AI复盘指定工单，分析处理是否规范、是否存在优化空间

### 2.2 非功能需求

- **稳定性**：支持高并发工单接入，流程流转不丢失、解析任务不堆积
- **可用性**：大模型调用异常、知识库检索失败有兜底策略，不影响工单正常流转
- **可扩展性**：支持新增工单类型、新增责任部门、新增知识库文档、新增流转规则
- **安全性**：工单数据脱敏处理，权限分级管控，防止敏感信息泄露
- **高性能**：AI解析任务异步处理，线程池隔离，不阻塞主业务流程

## 3. 整体架构设计

### 3.1 架构思想

采用**Java后端分层工程化 + Agent经典闭环架构**，遵循「思考解析→工具执行→流程流转→汇总总结」的Agent核心逻辑。全程Java后端主控，大模型仅负责智能识别、内容生成与分析，所有业务规则、流程流转、权限控制、数据持久化由后端掌控，兼顾智能化与企业级安全性、稳定性。

### 3.2 四层整体架构

#### 1）接入层（Controller）

接收前端工单提交、查询、复盘、知识库上传等请求，完成参数校验、权限校验，调度Agent核心服务，返回标准化结果与AI分析内容。

#### 2）Agent调度核心层（核心）

系统中枢模块，负责工单全生命周期智能调度：接收工单→AI智能解析→判断工单类型→匹配处理策略→执行自动办结/流程分派→记录结果→周期复盘总结，串联所有核心业务链路。

#### 3）工具能力层

封装所有可被AI调用的后端业务工具，实现能力可插拔：

- 工单智能解析工具（分类、分级、实体抽取）
- RAG知识库检索答疑工具
- 工单自动分派流转工具
- 重复工单匹配去重工具
- 工单统计与AI复盘报告生成工具

#### 4）基础服务层

包含LLM大模型客户端、轻量化向量检索服务、异步线程池、数据库持久化、权限控制、日志记录、定时任务、消息预警等基础能力。

### 3.3 核心业务流程（Agent完整闭环）

1. **工单接入**：用户/客服提交工单，包含问题描述、业务模块、图片、场景信息等
2. **AI智能解析**：Agent调用大模型，完成工单意图识别、优先级分级、关键信息抽取、重复工单校验
3. **策略决策**：Agent根据解析结果+业务规则，判断工单处理策略：自动办结/人工流转/紧急预警
4. **工具执行**：简单咨询类工单调用RAG知识库自动生成答复并办结；复杂工单自动分派责任人、进入流转流程
5. **流程监控**：监听工单处理状态，对超时未处理工单触发催办与预警
6. **结果持久化**：保存工单解析结果、处理记录、流转日志、AI答复内容
7. **周期AI复盘**：定时汇总所有工单数据，分析高频问题、故障根因、处理效率，生成复盘报告

## 4. 核心模块详细设计

### 4.1 LLM智能解析模块

统一封装大模型HTTP调用客户端，通过结构化Prompt强制模型输出标准化JSON结果，杜绝自由文本输出，保证后端可解析、可落地。

核心解析能力：

- 工单意图分类：咨询、故障、申诉、建议、运维报错、功能异常
- 优先级自动分级：紧急、高、中、低
- 关键信息抽取：业务模块、问题关键词、异常场景、用户诉求
- 工单复杂度判定：可自动办结/需人工处理

### 4.2 轻量化RAG知识库模块

适配工单答疑场景的轻量化知识库，无需重型向量框架，实现文档切片、语义检索、智能问答闭环。

- 支持FAQ文档、运维手册、业务说明、常见问题批量导入
- 文本切片、去重、关键词索引构建
- 根据工单问题语义匹配相似知识点，结合大模型生成精准答复
- 无匹配知识点时自动兜底，流转人工处理

### 4.3 工单智能分派与流转模块

结合AI解析结果+后台预设业务规则，实现自动化流转，替代人工分派：

- 按业务模块匹配对应责任部门与处理人
- 高优、紧急工单优先置顶、加急推送
- 超时工单自动标记、触发催办提醒
- 流转节点全程记录，可完整溯源工单处理链路

### 4.4 重复工单识别模块

通过文本相似度算法+AI语义判断，识别历史重复、相似工单：

- 避免同一问题重复建单、重复处理
- 关联历史工单处理结果，辅助当前工单快速解决
- 统计高频重复问题，为业务优化提供数据支撑

### 4.5 AI统计复盘模块

定时对周期内工单数据进行智能分析，摆脱人工统计报表：

- 工单总量、办结率、超时率、重复率数据统计
- 高频故障、高频咨询问题汇总
- 薄弱业务模块、高频报错节点分析
- 自动输出整改建议、优化方案与运维提示

### 4.6 异步任务与定时调度模块

基于Spring Schedule+自定义线程池实现任务隔离：

- AI解析任务异步执行，不阻塞工单提交主流程
- 定时触发每日/每周工单复盘统计
- 定时巡检未办结工单，触发超时预警

## 6. 核心接口设计

- **提交工单（AI自动解析）**：POST /agent/work/order/submit
- **工单列表查询**：GET /agent/work/order/list
- **工单详情+AI分析结果查询**：GET /agent/work/order/detail/{id}
- **知识库知识点新增/编辑**：POST /agent/work/knowledge/save
- **AI工单调优复盘**：POST /agent/work/order/review
- **获取周期复盘报告**：GET /agent/work/report/get
- **工单手动处理办结**：POST /agent/work/order/finish

## 7. 技术栈选型

- **核心框架**：SpringBoot 3.5.x
- **数据库**：MySQL 8.0
- **持久层**：MyBatis-Plus
- **文本处理**：Hutool、FastJSON2
- **任务调度**：Spring Schedule、自定义线程池
- **AI能力**：原生HTTP调用大模型（通义/星火/DeepSeek/Qwen通用适配）
- **轻量化RAG**：原生文本切片+语义相似度匹配，无重型框架依赖

## 8. 项目核心亮点（面试/简历重点）

- **纯原生Agent架构**：不依赖SpringAI、LangChain4j等框架，手写Agent解析、决策、执行、复盘完整闭环，彻底吃透AI Agent核心原理
- **业务场景高度落地**：基于企业真实工单运维、客服业务开发，区别于纯Demo项目，具备真实落地价值
- **AI+规则双驱动**：结合大模型智能推理+后端业务规则约束，解决纯AI不可控、纯规则不智能的行业痛点
- **轻量化RAG自研实现**：无需向量数据库，原生实现知识库问答，学习成本低、部署简单、适配中小企业场景
- **工程化能力完善**：异步任务、线程隔离、定时调度、数据统计、日志溯源、异常兜底全覆盖
- **降本增效价值明确**：AI自动办结简单工单，大幅减少人工重复工作，提升运维客服工作效率

## 9. 项目扩展方向

- 接入Redis实现会话持久化、高频工单缓存，提升查询性能
- 接入专业向量数据库，优化RAG检索精度，提升答疑准确率
- 集成钉钉/企业微信消息推送，工单异常、超时实时预警
- 增加工单AI根因溯源，统计系统bug、运营问题、用户操作问题占比
- 重构为SpringAI框架版本，适配企业框架化开发规范
- 新增工单评分体系，AI自动评估工单处理质量

## 10. 部署运行流程

1. 执行配套SQL脚本，创建工单、知识库、复盘报告三张业务表
2. 配置数据库连接、大模型API密钥、线程池参数、定时任务周期
3. 启动SpringBoot项目，导入基础FAQ知识库数据
4. 提交测试工单，验证AI解析、自动答疑、流程分派能力
5. 等待定时任务执行，查看AI自动生成的周期复盘报告

## 11. 项目总结

本项目是贴合企业业务的**AI Agent业务自动化核心实战项目**，兼顾Java后端工程化能力与AI智能决策能力，既包含数据库设计、异步调度、线程池管理、业务流程设计等传统后端核心能力，又包含LLM调用、语义解析、RAG知识库、Agent智能闭环等前沿AI技术。项目场景真实、技术亮点突出、业务价值清晰，可作为简历核心项目、面试重点讲解项目，同时可直接改造落地为企业内部智能工单运维系统。
